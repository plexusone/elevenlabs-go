## [Unreleased]

### Highlights

- WebSocket STT API migrated to scribe_v2_realtime with breaking changes to options and message formats

### Breaking

- `WebSocketSTTOptions` fields renamed: `SampleRate`/`Encoding` replaced with `AudioFormat`, `EnableWordTimestamps` replaced with `IncludeTimestamps`, `EnablePartials` removed (partials always enabled)
- `WebSocketSTTConnection.EndStream()` removed, use `Commit()` instead
- Default model changed from `scribe_v1` to `scribe_v2_realtime`

### Added

- `WebSocketSTTOptions.AudioFormat` field for specifying audio encoding (e.g., `pcm_16000`, `ulaw_8000`)
- `WebSocketSTTOptions.IncludeTimestamps` field for word-level timing
- `WebSocketSTTOptions.IncludeLanguageDetection` field for detected language in responses
- `WebSocketSTTOptions.CommitStrategy` field (`manual` or `vad`) for transcript commit control
- VAD (Voice Activity Detection) settings: `VADSilenceThresholdSecs`, `VADThreshold`, `MinSpeechDurationMs`, `MinSilenceDurationMs`
- `WebSocketSTTConnection.Commit()` method for manual transcript commit
- `WebSocketSTTConnection.SendAudioWithCommit()` method for sending audio with optional commit
- `WebSocketSTTConnection.SessionID()` method to retrieve server-assigned session ID
- OmniVoice STT provider conformance tests

### Changed

- WebSocket STT now uses query parameters for configuration instead of init message
- Audio data sent as `input_audio_chunk` message type with `audio_base_64` field
- Response message types: `partial_transcript`, `committed_transcript`, `committed_transcript_with_timestamps`
- OmniVoice conversion utilities updated for new STT options structure

### Documentation

- `TRD_STT.md` technical requirements document for WebSocket STT migration
