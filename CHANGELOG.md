## [v0.10.0] - 2026-04-04

### Highlights

- Agent Experience (AX) integration for intelligent error handling and retry logic
- OmniVoice provider enhancements with error classification and automatic retry
- New `ax/omnivoice.go` helper functions for seamless OmniVoice-core integration

### Added

- `omnivoice.Classifier` for ElevenLabs-specific error classification using AX metadata and HTTP status codes
- `omnivoice.DefaultRetryConfig()` and `omnivoice.RetryConfigWithCallback()` for configurable retry behavior
- TTS provider retry logic with exponential backoff, respecting error retryability
- `ax.CategoryForCode()` - Maps AX error codes to `resilience.ErrorCategory`
- `ax.IsRetryableCode()` - Checks if an AX error code is retryable
- `ax.SuggestionForCode()` - Returns helpful recovery suggestions for each error code
- `ax.OperationRequiredFields()` - Returns required fields for 10 ElevenLabs operations
- `ax.ToErrorInfo()` - Converts AX code to `resilience.ErrorInfo`
- `ax.ClassifyHTTPStatus()` - Classifies HTTP status codes to error categories
- `ax.AllCategories()` and `ax.CodesByCategory()` for error code discovery
- Integration tests with mock API server for retry scenarios
- Performance benchmarks for error classification and retry decisions

### Changed

- TTS provider now automatically retries on transient errors (429, 500, 502, 503) with exponential backoff
- TTS provider wraps all errors in `resilience.ProviderError` with classification metadata
- Auth errors (401, 403), validation errors (400, 422), and not-found errors (404) are not retried

### Dependencies

- Requires `github.com/plexusone/omnivoice-core` v0.8.0+ for resilience package

### Tests

- 8 integration tests covering retry scenarios (rate limit, auth, server errors, exhausted retries, context cancellation)
- 14 benchmarks validating performance targets (<1ms classification, <0.1ms retry decision)
- 7 classifier unit tests and 17 ax/omnivoice helper tests
